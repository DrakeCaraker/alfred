# Private Persona Intelligence

This directory stores per-user persona intelligence data. It is **gitignored** and never committed to the repository.

## What's stored here

`<user-hash>/persona-intel.enc` — An encrypted JSON file containing:
- Role description (free text from bootstrap)
- Persona fit assessment
- Categorized role and gap analysis
- Custom rule summary
- Persona suggestion history

This data is generated by `/persona analyze` and is a **derived artifact** — it can always be regenerated from your local onboarding state (`.claude/.onboarding-state.json`) and persona file (`.claude/alfred-persona.yaml`).

## Encryption

Files are encrypted at rest using AES-256-CBC (via `openssl`). The encryption key is stored separately at `.claude/.pilot-intel-key` (also gitignored, 0600 permissions).

**If you lose the key**, the encrypted files become unreadable — but the data can be regenerated by running `/persona analyze` again.

## Why encrypt?

The raw data may contain project descriptions or role details you'd rather not have exposed in backups, cloud sync (iCloud, Dropbox, Time Machine), or shared filesystems. Encryption ensures this data is unreadable outside your local environment.

## Decrypting manually

```bash
openssl enc -d -aes-256-cbc -pbkdf2 -in .pilot/private/<hash>/persona-intel.enc \
  -pass file:.claude/.pilot-intel-key
```

## Future use

When collective persona sharing (Phase 5) is available, this directory serves as the staging area for opt-in data submission. Only categorized, anonymized data would be shared — never the raw descriptions.
